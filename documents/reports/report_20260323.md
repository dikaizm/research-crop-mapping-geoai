## 1. Stage 2v3 — Incremental Top-K Channel Enumeration

### 1.1 Design

Stage 2v3 is a pure enumeration step — no U-Net training. It takes Stage 1v3 output
(`stage1v3_candidates.json`) and generates candidate channel sets for Stage 3 training.

Two phases per crop (using SI-ranked date and band lists from Stage 1v3):

**Phase A — Band sweep on rank-1 date:**
- Fix to the top-1 SI date per crop (highest separation date)
- For k = 1, 2, …, 9: take top-k bands × top-1 date → channel set
- Union across all 10 crops saved per k

**Phase B — Temporal sweep with all bands:**
- Use all 9 band candidates per crop (full SI-ranked list)
- For k = 1, 2, …, 10: take all bands × top-k dates → channel set
- Union across all 10 crops saved per k

## 2. Stage 3 — Exp C_v3 Wired

### 2.1 New Files

```
crop_mapping_pipeline/stages/experiments/exp_c_v3.py
```

`build_exp_C_v3_indices(phase, k, mmdd_to_date, local_band_to_idx, mlflow_run_id=None)`

- Experiment dispatch: `elif exp_key == "C_v3"` case added
- CLI: `C_v3` added to `--exp` choices, plus new args:
  - `--v3-phase band|date` (default: band)
  - `--v3-k INT` (default: 3)

### 2.3 Usage

