# Progress Report — 2026-03-24

## Summary

Pipeline infrastructure extended with three additions today:

1. **Loss versioning** — `PhenologyAwareLoss` separated from the default training path into an explicit `--loss-version v1/v2` system.
2. **Modular experiment registry** — if-elif chain replaced with `ExperimentConfig` dataclass + `build_registry()` supporting all current and future variants.
3. **Exp A_v2 added** — 4 individual phenological window baselines (Jan/Mar/Jul/Nov), each a separate 9-channel single-date model.

## 1. Loss Versioning (v1 / v2)

### 1.1 Problem

`PhenologyAwareLoss` was being applied silently as the default whenever B4 and B8 were present in the input (auto-detected via try/except). This created a confound: Exp A/B/C were trained before the loss was introduced, while later runs (Dv2) used it — making ablation comparisons unreliable.

### 1.2 Solution

Two versioned loss modules under `stages/losses/`:

| Version | Loss | Description |
|---|---|---|
| `v1` | `WeightedCrossEntropyLoss` | Standard weighted CE — default for all experiments |
| `v2` | `PhenologyAwareLoss` | Reduces weight to 0.1 for labeled crop pixels with NDVI < 0.3 (dormant/off-season) |

`PhenologyAwareLoss` requires B4 (red) and B8 (NIR) in the input tensor to compute NDVI. If B4/B8 are absent and `v2` is requested, the build function raises a `ValueError`.

### 1.3 Literature Support for v2 Threshold

The NDVI < 0.3 dormancy threshold is supported by:
- USDA IPAD documentation (off-season minimum for actively growing cropland)
- ISU/NASA technical reports on NDVI-based growing season detection

### 1.4 New Files

```
crop_mapping_pipeline/stages/losses/
├── __init__.py     ← exports build_loss_v1, build_loss_v2, PhenologyAwareLoss
├── v1.py           ← build_loss_v1(class_weights_tensor)
└── v2.py           ← PhenologyAwareLoss class + build_loss_v2(weights, band_names)
```

### 1.5 Changes to train_segmentation.py

- Removed inline `PhenologyAwareLoss` class
- `run_experiment()` now takes explicit `loss_version="v1"` param
- MLflow logs `loss: loss_v1` or `loss: loss_v2`
- CLI: `--loss-version v1|v2` (default: `v1`)

## 2. Modular Experiment Registry

### 2.1 Problem

The experiment dispatch in `main()` was a growing if-elif chain (~50 lines) that was hard to extend and impossible to introspect programmatically.

### 2.2 Solution

New `ExperimentConfig` dataclass + `build_registry()` in `stages/experiments/registry.py`:

```python
@dataclass
class ExperimentConfig:
    key:               str
    description:       str
    band_indices:      Any        # list[int] or per-year dict
    band_names:        list
    default_loss:      str  = "v1"
    mlflow_experiment: str  = MLFLOW_EXPERIMENT_TRAIN
    extra_kw:          dict = field(default_factory=dict)
```

`expand_exp_keys()` handles shorthands: `"A_v2"` → all `A_v2_*` keys, `"C_v3"` → all `C_v3_*` keys.

### 2.3 MLflow Experiment Separation

C_v3 variants now log to a separate MLflow experiment:

| Experiments | MLflow experiment name |
|---|---|
| A, A_v2, B, C, C_v2, C_v2_rf, D, D_v2 | `cropmap_segmentation_s2` |
| C_v3_* | `cropmap_segmentation_s2_v3` |

Constants in `config.py`:
```python
MLFLOW_EXPERIMENT_TRAIN    = "cropmap_segmentation_s2"
MLFLOW_EXPERIMENT_TRAIN_V3 = "cropmap_segmentation_s2_v3"
```

## 3. Exp A_v2 — Per-Phenological-Window Single-Date Baseline

### 3.1 Motivation

Exp A uses July 30 as the single reference date — chosen for data quality (99.6% valid pixels), not phenological optimality. A reviewer could argue that July 30 is a poor date for crops like Winter Wheat (already harvested by then). Exp A_v2 answers: *"what is the best achievable single-date result when the date is chosen optimally?"*

Each Exp A_v2 window is a separate 9-channel model trained on that window's date only — distinct from Exp B which combines all 4 dates into one 36-channel model.

### 3.2 Windows

| Key | Target | Selected date |
|---|---|---|
| `A_v2_Jan` | Jan-15 | nearest acquisition to Jan-15 |
| `A_v2_Mar` | Mar-15 | nearest acquisition to Mar-15 |
| `A_v2_Jul` | Jul-15 | nearest acquisition to Jul-15 (≈ Jul 30) |
| `A_v2_Nov` | Nov-15 | nearest acquisition to Nov-15 |

### 3.3 Expected Findings

- `A_v2_Mar` or `A_v2_Apr` should outperform `A` (Jul 30) for Winter Wheat
- `A_v2_Jul` should match Exp A closely (same window)
- Even the best `A_v2` window should underperform Exp B and Exp C

### 3.4 New Files

```
crop_mapping_pipeline/stages/experiments/exp_a_v2.py
```

`build_exp_A_v2_indices(local_date_to_idx, local_band_to_idx)` → `dict[label, (idx, names, date)]`

### 3.5 CLI

```bash
# Run all 4 windows + original Exp A
python stages/train_segmentation.py --exp A A_v2

# Run only A_v2 windows
python stages/train_segmentation.py --exp A_v2
```
