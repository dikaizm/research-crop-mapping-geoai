## 3. Preprocessing Pipeline (02_image_processing.ipynb)

### 3.1 Initial Version
**Correct ordering confirmed** (NoData must be assigned BEFORE resampling to prevent interpolation artifacts):

> **Interpolation artifacts**: When resampling is done before NoData assignment, invalid/missing pixels (e.g., fill values like 0 or 65535) are treated as valid data and blended with neighboring pixels by the resampling kernel. This produces erroneous intermediate values along the edges of valid regions. By assigning NoData *first*, the resampler ignores those pixels entirely, preserving clean boundaries between valid data and masked areas.

1. Assign NoData → Sentinel-2 (invalid pixels → -9999, float32)
2. Assign NoData → CDL (nodata=0)
3. Filter CDL classes (non-informative → background)
4. Resample CDL 30m → 10m (nearest neighbor)
5. Clip both rasters to intersection extent
6. Extract 9-band subset

### Fixes Applied
- `utils/label.py`: `label_filtering()` now correctly uses `.copy()` on profile and calls `profile.update(nodata=...)` before writing output.

### 3.2 Major Revision — CDL CRS Alignment (2026-03-02)

**Problem identified**: CDL is in EPSG:5070 (NAD83 / Conus Albers) while S2 is in EPSG:4326 (WGS84). The original notebook did not handle this CRS mismatch.

**Confirmed data properties:**

| Dataset | CRS | Resolution | Coverage |
|---|---|---|---|
| S2 (sacramento_3) | EPSG:4326 | ~10m | Study area only |
| CDL (2022_30m_cdls) | EPSG:5070 | 30m | Entire USA |

**New pipeline (complete rewrite):**

```
CDL (USA-wide, EPSG:5070, 30m)
  Step 1: Reproject EPSG:5070 → EPSG:4326  +  clip to study area  +  resample to ~10m
          → single rasterio.warp.reproject() call using S2 as reference grid
  Step 2: Filter CDL classes (keep target crops → rest to 0)

S2 samples (3 files, chronological: 2022-01-01, 2022-01-16, 2022-01-31)
  Step 3: Assign NoData per image (invalid/neg/NaN → -9999, cast float32)

Verify: CDL and all S2 outputs share same CRS, bounds, pixel dimensions
```

**Key design decision**: Steps 1 (reproject + clip + resample) are performed in a single `reproject()` call by passing the S2 image as the destination grid. This is more efficient than doing them separately and guarantees pixel-perfect alignment.

## 4. Novel Method: 3-Stage Band Selection

### Background
Literature review of DOI:10.3390/rs15215203 (RF importance filter for LSTM classification) confirmed the proposed approach is novel — **no published paper uses a segmentation model as the oracle in a sequential forward selection (SFS) loop** for multispectral Sentinel-2 crop mapping.

### Stage 1 — Filter Preselection
Reduces all input channels to top-25 candidates using:
- **GSI** (Global Separation Index): measures class separability per band
- **RF Importance**: mean decrease in Gini impurity
- **Joint Score**: `Score_b = α·GSI_norm + (1-α)·RF_norm` (α=0.5)

### Stage 2 — CNN Forward Selection (Novel Contribution)
- Oracle: lightweight U-Net (ResNet-18 encoder)
- Greedy sequential forward selection
- Accept band iff `mIoU_{t+1} > mIoU_t + δ` (δ=0.005)
- Stop: 3 consecutive rejections or max 15 bands
- Each iteration: 15 epochs, early stopping (patience=5)

### Stage 3 — Full Model Validation
Three experiments × Three models:
- **Exp A**: All bands (baseline)
- **Exp B**: Stage 1 filter only
- **Exp C**: Stage 2 CNN forward selection

Models: U-Net, DeepLabV3+ (with CBAM), SegFormer

### Code Changes
- `utils/band_selection.py`: Added `compute_rf_importance()`, `compute_joint_score()`, `select_top_k()`
- `notebooks/04_feature_analysis.ipynb`: Full 3-stage pipeline implementation

## 11. Optimal Class Selection for Model Training

**Files:** `documents/reports/cdl_class_selection_strategy.png`

### Selection Criterion: ≥1% of Study Area Coverage

**Recommendation: 11 cropland classes + background (class 0)**

### Scientific Justification

#### 1. Minimum Sample Size for Deep Learning
A class with ≥1% coverage occupies ≥2,038 ha in the study area. At 256×256 patches (10m resolution, ~510 ha per patch), this yields sufficient spatial coverage across hundreds of distinct agricultural fields for the model to learn generalizable spatial and spectral features. Classes below 1% (< ~2,000 ha) appear as sparse, fragmented polygons — typically fewer than 40 distinct fields — providing insufficient spatial diversity for CNN-based segmentation training. Studies on deep learning-based crop mapping consistently apply minimum area thresholds of 500–2,000 ha per class (e.g., Li et al. 2023, Tran et al. 2022).

#### 2. Class Imbalance Manageability
The selected set achieves a maximum imbalance ratio of 22.6:1 (Fallow 57,742 ha vs Grapes 2,549 ha). This is within the range addressable by standard techniques — inverse-frequency class weighting or Focal Loss — without causing training instability. Including classes below 0.5% would push this ratio above 100:1, at which point standard weighting becomes ineffective and rare classes are effectively ignored by the optimizer.

#### 3. Agricultural Significance of Sacramento Valley
The 11 classes represent the economically and agronomically dominant crops of the Sacramento Valley as documented by USDA NASS annual crop production statistics:
- **Fallow/Idle Cropland (61)**: essential for crop rotation monitoring and mapping agricultural intensity
- **Almonds (75) + Walnuts (76)**: California produces >80% of world almonds and >99% of US commercial walnuts; together they cover 420 km²
- **Tomatoes (54)**: Sacramento Valley produces ~95% of California's processing tomatoes
- **Rice (3)**: Northern Sacramento Valley is the primary California paddy rice growing region, with distinctive flooding-period spectral signature
- **Winter Wheat (24) + Sunflower (6)**: dominant annual grain and oilseed crops of the valley
- **Alfalfa (36)**: major perennial forage with a multi-cut phenological pattern distinct from other crops
- **Plums (220) + Grapes (69)**: regionally significant permanent crops with distinct canopy structure
- **Other Hay/Non Alfalfa (37)**: aggregated forage category covering 29 km²; retained as its area and phenology differ from alfalfa

#### 4. Reliable Evaluation Metrics
Per-class IoU (the primary metric) requires a statistically sufficient number of test samples per class. At a standard 80/20 train/test split, a class with <1% coverage provides fewer than 400 ha in the test set (~780 patches), making IoU estimates unreliable and high-variance. The selected 11 classes each contribute >500 ha to the test set, ensuring scientifically defensible per-class evaluation.
