#### Key New Section: Seleksi Fitur Spektral-Temporal

This section was entirely absent from the original literature review. It now covers:
- **Filter methods**: GSI (measures inter-class separability per band), RF feature importance (tree-based permutation sensitivity)
- **Wrapper methods**: Sequential Forward Selection (SFS) — evaluates feature subsets by training/testing a model
- **Hybrid approach**: Using Stage 1 filter scores to define the addition order for Stage 2 SFS, reducing complexity from O(N²) ≈ 210,000 evaluations to O(N) = 25 sequential evaluations
- Citations: rs12010162, rs15215203, alamimachichi2023

#### 4 Identified Research Gaps

1. Feature selection methods in existing studies (GSI, RF) are static filters — selection criteria are not optimized against the actual segmentation objective.
2. No study has used a CNN segmentation model as an oracle inside Sequential Forward Selection for multi-temporal satellite imagery.
3. Comparative studies of DeepLabV3+ and SegFormer specifically on multi-temporal Sentinel-2 crop mapping are limited.
4. There is no study combining a 3-stage filter+wrapper hybrid pipeline with both architectures under the same experimental conditions.

#### References Added

- `rs12010162`: "Optimizing Feature Selection of Individual Crop Types for Improved Crop Mapping", RS 2020, vol 12(1), p162. Added to `references.bib`.

### 5. Thesis Chapter Updates — Methodology (Stage 1→Stage 2 Clarification)

**File changed**: `documents/thesis/chapters/methodology.tex`

Updated Stage 2 and the experiment design sections to document the Stage 1 → Stage 2 relationship.

#### Stage 2 Description Update

Added explicit explanation that Stage 2 is NOT a pure greedy search (which would be O(N²) ≈ 210,000 evaluations for N=648) — instead, it evaluates subsets in the order defined by Stage 1, reducing complexity to O(N):

```
Stage 1 output: [B8_Jul30, B5_Mar17, B11_Jul30, ...] (ranked by joint score)

Stage 2 evaluation:
  Step 1: {B8_Jul30}                → mIoU = 0.42
  Step 2: {B8_Jul30, B5_Mar17}      → mIoU = 0.51
  Step 3: {... , B11_Jul30}         → mIoU = 0.58
  ...
  Step K*: accuracy plateaus → stop  → K* = optimal cut-off
```

**Stage 1 determines ORDER; Stage 2 determines K\* (how many to keep).**

| Exp C | Stage 2 output, K* bands | K* < 36 | Temporal + optimal selection |

- **A → B**: Measures whether temporal information improves accuracy over single-date.

### 7. geoai Library — `RasterPatchDataset` Added

**File changed**: `geoai/geoai/train.py`

Added `RasterPatchDataset` as a proper, generic class to the geoai library (inserted before `get_semantic_transform`, after the other Dataset/transform classes).

#### Motivation

The `SemanticSegmentationDataset` in geoai expects pre-chipped image files in a directory. For this project, pre-chipping 72 dates × ~378 patches × 11 bands is ~60 GB of disk — impractical for iteration. The `RasterPatchDataset` generates patches on-the-fly from rasters using rasterio windows, requiring zero pre-chipping.

The class was previously defined inline in `04_feature_analysis.ipynb`. Moving it to geoai makes it importable and reusable across notebooks.

#### Changes from Notebook Version → geoai Version

| | Notebook (project-specific) | geoai version (generic) |
|---|---|---|
| `keep_classes` | Global `KEEP_CLASSES` constant | Constructor parameter |
| `remap_lut` | Global `REMAP_LUT` array | Constructor parameter (optional) |
| Nodata value | Hardcoded `-9999.0` | `nodata` parameter |
| Docstring | None | Full NumPy-style docstring |

#### API

```python
from geoai.geoai.train import RasterPatchDataset

dataset = RasterPatchDataset(
    s2_paths=S2_PROCESSED,          # list of Sentinel-2 GeoTIFF paths
    cdl_path=CDL_FILTERED,          # CDL label GeoTIFF path
    patch_size=256,
    stride=256,                     # non-overlapping
    keep_classes=KEEP_CLASSES,      # CDL IDs considered valid foreground
    remap_lut=REMAP_LUT,            # CDL ID → model sequential ID mapping
    min_valid_frac=0.3,             # minimum labeled pixel fraction per patch
    band_indices=exp_A_idx,         # subset of stacked channels to select
    nodata=-9999.0,
)
# num_workers=0 required — rasterio handles cannot be pickled
```

### 8. Training Notebook — `12_train_segmentation.ipynb`

**File created**: `notebooks/12_train_segmentation.ipynb`

New notebook implementing Stage 3 full model training for all 6 experiment configurations (3 input setups × 2 architectures). Replaces the outdated `05_train_segmentation_model.ipynb` and `06–09` iteration notebooks.

#### Design

**Data loading** uses `RasterPatchDataset` (now in geoai) with on-the-fly patch generation — no pre-chipping required. Patches split 70/15/15 into train/val/test using `random_split` with a fixed seed.

**Training loop** uses geoai's `train_semantic_one_epoch` for the forward/backward pass, plus a custom `validate_one_epoch` that computes per-class mIoU (background excluded) and overall accuracy.

**Loss** uses `nn.CrossEntropyLoss` with inverse-frequency class weights computed directly from the CDL raster — handles the 22.6:1 imbalance between Fallow and Grapes.

**Optimiser/scheduler** follow the thesis methodology: AdamW + PolynomialLR(power=0.9), with architecture-specific learning rates:

| Architecture | LR | Weight Decay |
|---|---|---|
| DeepLabV3+CBAM | 1e-4 | 1e-4 |
| SegFormer | 6e-5 | 1e-2 |

**Early stopping**: patience = 10 epochs on val mIoU.

#### Band Configuration per Experiment

| Exp | Bands selected | Channels | Selection logic |
|---|---|---|---|
| A | 9 vege bands (B2–B12 excl. B1, B8A) from Jul 30 | 9 | Calendar (best coverage date) |

| C | K* bands from `forward_selection_history.csv` | K* | Stage 2 CNN forward selection |

The `VEGE_BANDS` list (`["B2","B3","B4","B5","B6","B7","B8","B11","B12"]`) is applied as `band_indices` into the stacked multi-date array, matching the 9-channel / 36-channel figures in the thesis.

If a cell calling `mlflow.start_run()` is re-executed (e.g., after a kernel restart) without first running `end_run()`, the previous run stays stuck as RUNNING. Fixed by adding defensive cleanup at the start of Stage 1 (cell `fa000012`) and Stage 2 (cell `fa000026`):

If an exception occurs between `start_run()` and `end_run()`, the run stays RUNNING. Fixed by wrapping the logging block in `try/finally`:

```python
try:

    mlflow.end_run(status="FINISHED")

    mlflow.end_run(status="FAILED")
```

## 13. Full-Scale Notebook Preparation — Multi-Year Dataset

Updated all 4 notebooks to support the full-scale production run using the complete 3-year dataset (2022+2023 for training, 2024 for testing).

### Training Approach Decision

**Chosen**: Direct combined training on 2022+2023 (no transfer learning).

**Rationale**:
- More diverse training data — combining two growing seasons exposes the model to inter-annual variability in crop spectral signatures
- CDL label schema is consistent across years (same USDA NASS methodology)
- Simpler pipeline — no stage-gate (pre-train → fine-tune) means fewer hyperparameters and cleaner MLflow logging
- Transfer learning would not add value here since the source and target domains share the same sensor, label schema, and study area

### Scope Decisions

| Notebook | Scope | Rationale |
|---|---|---|
| `02_image_processing.ipynb` | All 3 years (2022, 2023, 2024) | CDL must be pre-processed for every year used in training or testing |
| `03_image_analysis.ipynb` | All 3 years | EDA is not data leakage; understanding the test-year (2024) distribution is needed for sound analysis |
| `04_feature_analysis.ipynb` | 2022 only | K* bands are year-agnostic — spectral separability (GSI, RF) is physically determined by crop spectral signatures, which converge with a single full year (~25M pixels in 2022). Re-running per year is unnecessary and expensive. |
| `12_train_segmentation.ipynb` | Train: 2022+2023; Test: 2024 | Temporal honesty — test year is fully held out; model never sees 2024 data during training or validation |

### Test Strategy

**Year-based split**: all 2024 patches used as test set (not a random subset of the full dataset). This provides:
- Temporally honest evaluation — no future data leaks into training
- Consistent test set across all 6 experiment runs
- Exp A uses the `2024-07-29` image as the single representative date (closest available to peak-season Jul 30)
